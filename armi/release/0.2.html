ARMI v0.2 Release Notes¶

ARMI v0.2.4¶

Release Date: TBD

What’s new in ARMI¶

  1. Added neutronics settings: inners and outers for downstream support.

  2. Removed unused Thermal Hydraulics settings.

  3. Minor code re-org, moving math utilities into their own module.

  4. Removed the PyYaml dependency.

  5. Removed all bare import armi statements, for code clarity.

  6. Replaced setting stationaryBlocks (now deprecated) with stationaryBlockFlags setting (PR#665)

  7. Renamed control rod assembly parameters for generic implementations of control rod handling.

  8. Changed the default Git branch name to main.

  9. Changed the default value of the trackAssems setting to False.

  10. Removed _swapFluxParam method (PR#665)

  11. Added new UserPlugin functionality.

  12. Made the min/max temperatures of Material curves discoverable.

  13. Introduction of axial expansion changer to enable component-level axial expansion of assemblies.

  14. Add setting inputHeightsConsideredHot to enable thermal expansion of assemblies at BOL.

  15. Add blueprints setting, axial expansion target component, to enable users to manually set a “target component” for axial expansion.

  16. Allow database comparison to continue even if unpackSpecialData fails.

  17. Code clarity rewrite of armi/cases/case.py::copyInterfaceInputs.

  18. Added setting nonUniformAssemFlags that allows specific assems to not conform to uniform mesh. (PR#879 and PR#827).

  19. Added reporting of neutron and gamma energy groups in the XS library __repr__ for clearer tracking of which cross sections are applied to the core state.

  20. Removed quiet kwarg from geometry and block converters as this was generally unused in the code base and was only implemented for developmental debugging.

  21. Refactored the UniformMeshGeometryConverter to allow for mapping of number densities and parameters to and from a single assembly rather than requiring an entire core.

  22. Updated NHFLUX reader to store VARIANT data that was being discarded. Reading/writing VARIANT NHFLUX files now show binary equivalance.

  23. The newReports were moved into their final location in armi/bookkeeping/report/.

  24. Refactored the UniformMeshGeometryConverter again to auto-detect if block-level parameters are scalar or array values and implemented a feature to optionally disable number density mapping.

  25. Remove ability for users to use separate blueprints in snapshot runs. They are only determined from database. (PR#872 https://github.com/terrapower/armi/pull/872)

  26. Removed the last usage of settingsRules, so that everything can use settingsValidation now.

  27. TBD

Bug fixes¶

  1. Fixed issues finding ISOXX files cross-platform.

  2. Fixed issue with docs not correctly loading their Git submodule in TOX.

  3. Multiple bug fixes in growToFullCore.

  4. Block.getWettedPerim was moved to HexBlock, as that was more accurate.

  5. pathTools.cleanPath() is not much more linear, and handles the MPI use-case better.

  6. Fixed bugs in the ARMI memory profiler.

  7. Fixed linear expansion in Alloy200.

  8. Remove copy.deepcopy from armi/reactor/converters/uniformMesh.py

  9. Clarify docstring for armi/reactor/components/complexShapes.py::Helix

  10. Bug fix in armi/reactor/components/complexShapes.py::Helix::getCircleInnerDiameter

  11. Fixed issue with axial expansion changer and axial meshing in armi/reactor/reactors.py::Core::processLoading.

  12. Bug fix and clarity improvements for how number densities are initialized for components.

  13. Bug fix in armi/cases/case.py::copyInterfaceInputs

  14. Bug fix in armi/reactor/components/component.py::getReac

  15. Bug fix in armi/reactor/converters/uniformMesh.py to prevent _mapStateFromReactorToOther from clearing out unchanged parameter data.

  16. Migration of the axialMesh core parameter from the neutronics category to be general to prevent inconsistencies in the axial core mesh and the assembly meshes.

  17. Store thermally expanded block heights at BOL (via b.p.heightBOL) when thermally expanding assemblies in armi/reactor/reactors.py::Core::processLoading.

  18. Fixed bug where components were different if initialized through blueprints vs init (bug active for ~ 2 months).

  19. Fixed bug where component mass was being conserved in axial expansion instead of correct density (PR#846)

  20. Fixed bug in armi/reactor/blocks.py::HexBlock::rotatePins that failed to modify pinLocation parameter (#855).

  21. Fixed bug in armi/reactor.py::Core::_applyThermalExpansion that failed to call the block.completeInitiaLoading (#885).

  22. Fixed bug where a settings validator would complain that both simple and detailed cycles inputs were being used even if only the detailed input was entered.

  23. TBD

ARMI v0.2.0¶

Release Date: 2021-11-19

The API has started to solidify, and the number of external-facing changes have started to slow down. This release is a stake in the ground on a stable API.

What’s new in ARMI v0.2.0¶

  1. Made user settings immutable to avoid confusing runtime behavior

  2. Removed the concept of ‘facemaps’ (now replaced with more general grids)

  3. Added ability to use module-level logging for more precise debugging

  4. Added ability to write full tips-up hex asciimaps

  5. Fixed ability to serialize grid blueprints

  6. Improved code coverage and linting

  7. Added a latin hypercube suite builder for parameter sweeps

  8. Added several clarifications, fixes, and updates to documentation

  9. Updated units labels on several parameters

  10. Added protections against deleting directories

  11. Updated spontaneous fission data

  12. Removed confusing Charge Fuel Pool from core

  13. Sped up YAML reading

  14. Removed localization module

  15. Added ANL116 energy group structure

  16. Added setting to control auto-creation of within-block grids

  17. Added new plot/summarizing capabilities

  18. Added ability for GUI to save map as image

  19. Added C5G7 compositions and dimensions to LWR tutorial

  20. Added 1d/2d mesh reading/writing to GEODST

Backwards incompatible changes¶

There may be some new errors based on updated input checking.

Bug fixes¶

  1. Fixed centering of full-symmetry Cartesian lattice maps

  2. Fixed issues with grids that had multii-index locations

  3. Removed test files from coverage check

  4. Fixed order of operations issue in rotatePins

  5. Fixed incorrect multiplicity for non-grid block components

  6. Many additional bugfixes and cleanups (see PR list)
